Apache ECharts
交互式可视分析
演示

把关系表达、视觉映射与多图联动收进一个统一界面。

场景 数据可视化课程汇报
载体 知识关系交互看板
核心信号 交互优先
Graph · Linked Views

静态图能展示结果，
交互图才能支持探索

静态图的问题

  • 只能看结果，难以看清分析过程
  • 单张图很难同时表达关系、属性和局部结构

这个 Demo 的目标

  • 让观众在汇报中直接点击、筛选和聚焦
  • 用一个统一界面讲清 ECharts 的综合能力

技术选型

为什么是 ECharts

Graph

关系结构天然可见

知识点之间的前置、关联和层次可以直接被看见。

Dashboard

交互分析更自然

多图联动、动态更新和组件协同都能低成本成立。

Graph + VisualMap + Events + setOption，让 ECharts 更像交互分析界面，而不只是图表库。

差别不在能不能画图，
而在能不能让人探索

Matplotlib

更像结果输出

  • 擅长静态结果和论文插图
  • 交互、联动和下钻都不是默认能力

Apache ECharts

更像分析界面

  • 天然适合浏览器中的交互式展示
  • 点击、筛选和联动都可以直接成为体验的一部分

这次汇报要展示的是探索过程，而不是结果截图。

这个 Demo 重点展示四件事

Graph

关系结构一眼可见。

VisualMap

属性直接映射为视觉。

Connect

主图与辅图同步联动。

State

筛选与切换即时更新。

核心演示

交互式知识关系分析看板

Live Interactive Linked Views

看三件事

  • 点击节点：进入局部分析
  • 点击辅图：反向驱动主图
  • 颜色与大小：同时编码难度和重要度
